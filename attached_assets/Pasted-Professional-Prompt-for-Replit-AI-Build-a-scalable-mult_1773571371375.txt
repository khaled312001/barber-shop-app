Professional Prompt for Replit AI

Build a scalable multi-tenant SaaS barber shop management system called Barmagly.

The application must support three main roles:

Super Admin (Platform Owner – Barmagly)

Salon Admin / Manager (Store Owner)

Staff / Barber

The system must allow multiple salons (stores) to operate inside the same platform, where each salon has its own data, users, bookings, and settings.

The platform owner (Super Admin) can manage all salons from a central dashboard.

Architecture Requirements

Use a modern scalable architecture suitable for SaaS platforms.

Suggested stack:

Frontend
React / Next.js

Backend
Node.js (Express) or Laravel API

Database
PostgreSQL or MySQL

Authentication
JWT or session-based authentication

Multi-tenancy strategy
Tenant ID linked to each salon/store

Authentication System

The app must allow login from the same application but assign different permissions based on role.

Roles:

Super Admin
Salon Admin (Store Manager)
Staff / Barber

Permissions:

Super Admin

Access the full platform

Manage all salons

Manage subscriptions

Control platform analytics

Salon Admin

Manage their own store only

Manage barbers, services, bookings

View store analytics

Staff / Barber

View their schedule

Manage appointments assigned to them

Multi-Tenant System

Each salon must be treated as a tenant.

Each tenant should have:

unique store ID

store name

store settings

store staff

store services

store bookings

store customers

store subscription

All data must be isolated per tenant.
